# Release notes — v0.1.0

**Release date:** 2026-06-25

The first tagged release of the Organizational Memory System: a local,
deterministic toolkit that turns meeting text into structured organizational
memory. It performs no network calls and uses no external models or embeddings.

## Summary

v0.1.0 delivers the complete pipeline — ingestion, extraction, storage, recall,
analytics, and reporting — plus a full command-line interface, deterministic
demos, observability and validation tooling, and a documentation set suitable for
review.

## Major capabilities

- **Extraction.** Rule-based, deterministic extraction of decisions,
  commitments, tasks, open loops, participants, risks, action items,
  dependencies, and topics.
- **Storage.** A `MemoryStore` interface with interchangeable JSON and SQLite
  backends, plus snapshots and migrations.
- **Recall.** A compact query parser (`key:value` filters + free text) and
  lexical keyword search with coverage scoring and an exact-phrase bonus.
- **Analytics.** Memory health, bottlenecks, owner load, decision velocity,
  commitment completion, open-loop metrics, overdue tasks, and productivity.
- **Reports.** Typed report builders (meeting, weekly, follow-up, decisions,
  commitments, open-loops, organizational-memory) with Markdown/JSON/CSV
  exporters.
- **CLI.** A complete `organizational-memory` command-line interface.
- **Hardening.** Structured logging, local error reporting, configuration
  validation, coverage/benchmark/load/stress/fuzz tooling, and release
  validation.

The full test suite is deterministic and runs locally with no network access.

## Commit count target

The repository reaches exactly **225 commits** at v0.1.0, built across ten

reports, CLI & demos, hardening, and release & showcase).

## Known limitations

- Local-only: no server, API, authentication, or multi-user isolation.
- Plaintext storage: store files are not encrypted by the toolkit.
- Lexical recall: no semantic search, synonyms, or fuzzy matching.
- Rule-based extraction: relies on cues like "we decided", "I will", "TODO:",
  "Risk:", and question marks.
- Time-relative metrics depend on the `--now` value you supply.

## Upgrade notes

This is the first tagged release, so there are no prior versions to upgrade from.
Store files written during development remain compatible with the v0.1.0 store
formats.
